Sistema de Afastamentos e Solicitações
Visão geral

Sistema responsável por gerenciar solicitações de afastamento de funcionários, incluindo atestados médicos, férias e afastamentos relacionados ao INSS.

Arquitetura do Sistema
Arquitetura em Camadas

O sistema foi estruturado utilizando Arquitetura em Camadas, separando responsabilidades para melhor organização e manutenção.

Camadas utilizadas:
1. Camada de Apresentação (Controller)
Responsável por receber requisições
Valida entrada básica
Encaminha para a camada de serviço
Retorna respostas ao cliente
2. Camada de Serviço (Service)
Contém regras de negócio
Executa validações principais
Aplica lógica de RBAC
Orquestra operações entre domínio e banco
3. Camada de Domínio (Domain)
Define entidades e regras centrais
Contém enums e validações básicas
Mantém regras independentes de infraestrutura
4. Camada de Persistência (Repository / ORM)
Responsável pelo acesso ao banco de dados
Utiliza ORM (SQLAlchemy)
Executa operações CRUD
Segurança do Sistema
Autenticação

O sistema utiliza JWT (JSON Web Token) para autenticação.

Fluxo:

Usuário realiza login com email e senha
Sistema valida credenciais
Gera token JWT
Token é usado em requisições subsequentes

Características:

Stateless
Identificação do usuário via token
Validação via middleware/service
Senhas (Hash)

Senhas são armazenadas utilizando hash.

Utiliza algoritmo de hash (ex: bcrypt)
Comparação feita no login
Senhas nunca ficam em texto puro no banco
Autorização (RBAC)

Após autenticação via JWT:

Role é extraído do token
Sistema verifica permissões antes de executar ações
Controle baseado em Role-Based Access Control
